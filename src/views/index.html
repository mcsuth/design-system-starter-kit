What is the Design System Starter Kit?

The starter kit is a rapid prototyping environment using the Salesforce Lightning Design System, HTML, and Sass. It‚Äôs built for designers, developers, and anyone who wants to build a prototype and dabbles in HTML.

Get to know the starter kit

Now you‚Äôve installed the starter kit locally, let‚Äôs have a look at what‚Äôs inside. The kit‚Äôs file structure looks like this:

Deploying on Heroku

You‚Äôve pushed your work to your fork (git push origin master), and now it‚Äôs now time to share your prototype with the world!

Deploying your fork to Heroku is free and only takes 2 minutes.

  1. Head to your fork on GitHub
  2. Click the `Deploy to Heroku` button
  3. In DevTools, go to the `Sources` tab
  4. Submit the form (Deploy for Free button)
  5. Click the `View` button

That‚Äôs it! You can now share the link with the rest of the world!

Linking to GitHub

Now that you‚Äôve got your project deployed, what do you do with any changes thereafter?

Heroku has a really cool feature that integrates with GitHub and does automatic deployments of every pushed commit: https://devcenter.heroku.com/articles/github-integration

  1. Go to `Manage App`
  2. `Deploy`
  3. `GitHub`
  4. `Connect to GitHub`
  5. Authorize if you need to
  6. Search `design-system-starter-kit and connect`
  7. Select `Enable Automatic Deploys`

This means that every time you push a change, your application will be deployed.

So if you edit something and it gets committed and pushed to the master branch, a deploy hook will get triggered on Heroku and after a few minutes your application will automatically update!
